# Design System: My Alaska Ocean — ทะเลอลาสก้าของหนู

## 1. Overview

**Creative North Star: "สนามเด็กเล่นที่โตไปกับหนู" (The Playground That Grows With You)**

นี่ไม่ใช่ดีไซน์ของเกมเดียว แต่เป็นระบบของ "สนามเด็กเล่นดิจิทัล" ที่จะมีหลายโลก หลายธีม สำหรับเด็ก 1-15 ปี — **ทะเลอลาสก้าคือโลกแรก** ไม่ใช่ข้อจำกัด สิ่งที่คงที่ข้ามทุกโลกคือบุคลิก: สดใส ขี้เล่น มีชีวิตชีวา ทุกอย่างที่กดได้ต้อง "นุ่มนิ่มบีบได้" เหมือนของเล่นยางนุ่ม และฉากต้องเป็นโลกที่มีชีวิต — ตัวละครว่าย/ลอย/กะพริบตาตลอดเวลา — บนพื้นหลังที่สงบพอให้เด็กเล็กโฟกัสสิ่งเดียวได้

ระบบนี้ปฏิเสธสามสิ่งอย่างชัดเจน (จาก PRODUCT.md): ความฉูดฉาดจนล้า, หน้าตาการ์ตูนสำเร็จรูปแบบ clipart, และหน้าตาแอปเรียนจริงจังแบบแบบฝึกหัด ความมีชีวิตชีวามาจากอนิเมชันนุ่ม ๆ ของตัวละคร ไม่ใช่จากความรกหรือสีแสบตา และการเรียนรู้ต้องซ่อนอยู่ในการเล่นเสมอ

**Key Characteristics:**
- พื้นหลังคือ "โลก" ของธีมนั้น (ทะเล = ไล่เฉดน้ำ 5 ขั้นจากผิวสู่ก้นทะเล) — ลึก สงบ เป็นเวที ไม่ใช่ตัวแสดง
- องค์ประกอบกดได้เป็นสีขาวสะอาด มุมโค้งใหญ่ บีบยุบเมื่อกด (squash & stretch)
- ตัวละคร SVG สไตล์ "D": สัดส่วนอิงของจริง + สีเรียบ + ตาโตอ่านอารมณ์ง่าย
- ไม่มีตัวหนังสือเป็นเงื่อนไขการเล่น — ข้อความมีไว้เพื่อผู้ปกครอง
- เป้าหมายแตะทุกชิ้น ≥80px (ยกเว้นปุ่มกลับที่ "เล็กโดยตั้งใจ" 56px กันเด็กกดโดน)

## 2. Colors: The Alaska Water Column

จานสีของโลกทะเล: เสาน้ำ 5 ระดับเป็นพื้น สีขาวน้ำแข็งเป็นพื้นผิวกดได้ และสีพาสเทลอิ่ม 4 สีเป็นเอกลักษณ์ประจำเกม

### Primary
- **Foam White** (#ffffff → ไล่ลง Ice Mist #e8f2f8): พื้นผิวของทุกสิ่งที่กดได้ — ปุ่ม การ์ดเกม ตัวเลือก ความสะอาดของมันคือสิ่งที่บอกเด็กว่า "อันนี้แตะได้"
- **Ink Deep** (#13496e): ตัวหนังสือบนพื้นขาว สีเดียวกับน้ำลึก ทำให้ตัวอักษรรู้สึกเป็นส่วนหนึ่งของโลกทะเล

### Secondary — สีประจำเกม (game identity accents)
- **Glacier Blue** (#79b8d6): เกมแตะทะเล (1-2 ปี)
- **Sunset Orange** (#f2a25c): เกมจับคู่เงา (2-4 ปี)
- **Kelp Green** (#6ec99a): เกมนับสัตว์ทะเล (3-5 ปี)
- **Anemone Purple** (#c98ad9): เกมเสียงเรียกใคร (3-5 ปี)
- **Sun Gold** (#f7d154): สี accent ของเกมวาดตามรอย และสีที่ห้าของชุดคอนเฟตตี
- **Coral** (#ef7d6e): สี accent ของเกมสะกดคำ
- **Teal** (#2fb6a8): สี accent ของเกมความจำจับคู่
- ใช้เป็นขอบล่างหนา 6px ของการ์ดเกมบน hub และเป็นสีนำของฉากในเกมนั้น — หนึ่งเกม หนึ่งสี ห้ามสลับ
- **Source of truth ในโค้ด:** ทุกสี UI ถูกประกาศเป็น CSS custom properties บน `:root` ใน `css/main.css` ชื่อตรงกับเอกสารนี้ — โค้ดใหม่ต้องใช้ `var(--token)` ห้าม hardcode hex (ยกเว้นสีงานศิลป์สัตว์ใน animals.js)

### Neutral — เสาน้ำ (background world)
- **Sky Foam** (#bfe6f2): ผิวน้ำรับแสง — ขอบบนสุดของ gradient
- **Shallow Water** (#6db7d8) → **Mid Water** (#2f86b8) → **Deep Water** (#155e8d): ช่วงกลางของเสาน้ำ
- **Abyss** (#0d3f63): ก้นทะเล — ขอบล่างของ gradient และพื้นเต็มจอของ overlay (เช่น คำแนะนำหมุนจอ)
- **Glow Pale** (#dceefb): ข้อความรองบนพื้นน้ำเข้ม (เช่น ป้ายแจ้งผู้ปกครอง)
- **Slate Muted** (#3f6a87): ป้ายกำกับรอง (ช่วงวัย) บนพื้นขาว — เข้มพอให้ผ่าน 4.5:1 บน Ice Mist

### Named Rules
**The Calm Water Rule.** พื้นหลังเป็นไล่เฉดของโลกธีมเสมอ ห้ามใส่ลวดลาย/สีฉูดฉาดลงพื้นหลัง ความสดใสทั้งหมดเป็นสิทธิ์ของตัวละครและองค์ประกอบกดได้เท่านั้น — ถ้าจอดูวุ่นวายก่อนที่อะไรจะขยับ แปลว่าผิดกฎนี้แล้ว

**The One Game One Color Rule.** แต่ละมินิเกมมีสี accent ประจำตัวหนึ่งสี ใช้ซ้ำกันทุกที่ที่อ้างถึงเกมนั้น โลกธีมใหม่ในอนาคตให้ตั้งชุด accent ของตัวเองด้วยความอิ่มสีระดับเดียวกัน (พาสเทลอิ่ม ไม่ใช่นีออน)

## 3. Typography

**Display/Body Font:** system-ui (-apple-system, "Segoe UI", sans-serif)

**Character:** ฟอนต์ระบบตัวหนา (800) มุมมนตามธรรมชาติของ system-ui — อ่านง่าย โหลดทันที รองรับไทย/อังกฤษเท่ากันทุกเครื่อง ความขี้เล่นมาจากน้ำหนักที่หนาและขนาดที่ใหญ่ ไม่ใช่จากฟอนต์แฟนซี

### Hierarchy
- **Display** (800, clamp(18px, 3.5vw, 30px)): ชื่อเกมบนแถบบนสุด — Ink Deep พร้อม halo ขาวจาง (0 1px 0 rgba(255,255,255,.35)) เพราะส่วนบนของเสาน้ำสว่างเกินกว่าตัวหนังสือขาวจะอ่านออก
- **Headline** (800, clamp(16px, 2.6vw, 24px)): ชื่อมินิเกมบนการ์ด hub — Ink Deep บนพื้นขาว
- **Body** (400, 22px, line-height 1.6): ข้อความถึงผู้ปกครอง (overlay, คำแนะนำ) — ขาวบนพื้นน้ำเข้ม
- **Label** (400, clamp(11px, 1.6vw, 15px)): ป้ายช่วงวัยใต้ชื่อเกม — Slate Muted

### Named Rules
**The No-Reading Rule.** ผู้เล่นหลักอ่านหนังสือไม่ออก — ตัวหนังสือทุกชิ้นต้องเป็น "ของแถม" สำหรับผู้ปกครอง ห้ามมีขั้นตอนการเล่นใดที่ต้องอ่านจึงจะผ่านได้ ทุกคำสั่งต้องมีเสียงพูดคู่เสมอ

**The Both-Languages Rule.** ข้อความที่เด็กเห็นบ่อยควรมีทั้งไทยและอังกฤษ (เกมนี้คือเครื่องมือสองภาษา) — ผ่านระบบ i18n เสมอ ห้าม hardcode ภาษาเดียว

## 4. Elevation

ระบบใช้เงาเดี่ยวนุ่มเพียงแบบเดียว ความลึกที่เหลือเล่าผ่านสีของเสาน้ำ (สว่างบน-มืดล่าง) ไม่ใช่ผ่านเงาซ้อนหลายชั้น

### Shadow Vocabulary
- **Soft Lift** (`box-shadow: 0 6px 16px rgba(0,20,40,.3)`): เงาเดียวของระบบ ใช้กับทุกองค์ประกอบกดได้ — บอกว่า "ชิ้นนี้ลอยอยู่เหนือน้ำ หยิบได้"
- **Text Halo** (`text-shadow: 0 1px 0 rgba(255,255,255,.35)`): halo ขาวจางใต้ตัวหนังสือ Ink บนผิวน้ำสว่าง; ตัวหนังสือขาวบนน้ำลึก (เช่นป้ายแจ้งผู้ปกครอง) ใช้เงาเข้ม 0 1px 3px rgba(0,30,60,.6)

### Named Rules
**The Soft Lift Rule.** เงามีโทนน้ำเงินเข้ม (rgba(0,20,40,…)) เสมอ ไม่ใช่ดำล้วน และมีแค่ระดับเดียว — ถ้าอยากให้บางอย่างเด่นขึ้น ให้ขยับ/ขยายมัน อย่าเพิ่มเงา

## 5. Components

ทุกองค์ประกอบกดได้มีบุคลิกเดียวกัน: **นุ่มนิ่มบีบได้** — เหมือนของเล่นยางนุ่ม กดแล้วยุบ ปล่อยแล้วคืนตัว

### Buttons
- **Shape:** มุมโค้งใหญ่ใจดี (22px) ขั้นต่ำ 80×80px
- **Primary:** ไล่เฉดขาวลงฟ้าน้ำแข็ง (#ffffff → #e8f2f8) ตัวหนังสือ Ink Deep + Soft Lift shadow
- **Active (กด):** บีบยุบ `scale(.93)` ภายใน .12s ease — นี่คือหัวใจของความ "บีบได้" ห้ามตัดออก
- **Round (ปุ่มเสียง/ปุ่มกลับ):** วงกลม 56px — เล็กกว่ามาตรฐาน 80px โดยตั้งใจ เพราะเป็นปุ่มของผู้ปกครอง ไม่ใช่ของเด็ก

### Game Tile (การ์ดเกมบน hub)
- ปุ่ม Primary ขนาดใหญ่เต็มช่อง grid 2×2 + ตัวละครประจำเกม (SVG 150px ลอยขึ้นลง) + ชื่อเกม (Headline) + ช่วงวัย (Label)
- **ขอบล่างหนา 6px** สี accent ประจำเกม — นี่คือที่เดียวที่สี identity ปรากฏบน hub

### ตัวละคร (Signature Component)
- SVG วาดในโค้ด (ฟังก์ชันคืน element) สไตล์ "D": สัดส่วนอิงสัตว์จริง สีเรียบ เงา 1-2 ชั้น ตาโตอ่านอารมณ์ง่าย
- ทุกตัวมีวงจรชีวิต: ลอย (`float` 3.2s สลับขึ้นลง ±8px เอียง ±1.5°), กะพริบตา (`blink` ทุก 4s), ตอบสนองเมื่อแตะ (`boing` squash-stretch .55s), ปรากฏตัว (`pop-in` .45s เด้งเกินแล้วคืน), ส่ายหัวเบา ๆ เมื่อยังไม่ถูก (`shake` .5s — นุ่ม ไม่มีสีแดง ไม่มีเสียงตำหนิ)

### Celebration
- คอนเฟตตี (สี่เหลี่ยม 14px มุม 3px ร่วงหมุน 1.6s) + เสียงเชียร์สังเคราะห์ + พูดชื่อซ้ำสองภาษา — รางวัลเดียวของเกม
- ฟองอากาศประดับ (วงโปร่งขอบขาว 40% ลอยขึ้น) ให้โลกหายใจตลอดเวลา

### Named Rules
**The Squeeze Rule.** ทุกสิ่งที่กดได้ต้องตอบสนองการกดด้วยการบีบยุบ (scale .93) และตัวละครต้องเด้งแบบ squash & stretch — การตอบสนองทางกายภาพคือภาษาหลักที่เด็ก 1 ขวบเข้าใจ

**The 80px Rule.** เป้าหมายแตะของเด็กขั้นต่ำ 80×80px ไม่มีข้อยกเว้น ปุ่มที่เล็กกว่านั้นแปลว่า "ปุ่มนี้ของผู้ปกครอง"

## 6. Do's and Don'ts

### Do:
- **Do** ให้ทุกองค์ประกอบกดได้บีบยุบเมื่อกด (`:active { transform: scale(.93) }`) และมีเงา Soft Lift เดียวกัน
- **Do** ให้ตัวละครขยับตลอดเวลา (ลอย กะพริบตา) — โลกที่หยุดนิ่งคือโลกที่ตายสำหรับเด็กเล็ก
- **Do** จับคู่ทุกข้อความสำคัญกับเสียงพูด TTS สองภาษาผ่าน i18n/audio ที่มีอยู่
- **Do** ฉลองความสำเร็จด้วยคอนเฟตตี + เสียงเชียร์ + พูดชื่อซ้ำ — และฉลองบ่อย ๆ
- **Do** ใช้สี accent ประจำเกมเดิมทุกครั้งที่อ้างถึงเกมนั้น (One Game One Color)
- **Do** เผื่อโลกธีมใหม่: พื้นหลัง = ไล่เฉดของโลกนั้น, ปุ่ม/การ์ด = ขาวสะอาดเหมือนเดิมทุกโลก

### Don't:
- **Don't** "ฉูดฉาดจนล้า" — ห้ามสีนีออนแสบตา ห้ามกะพริบรัว ห้ามอนิเมชันวูบวาบทั้งจอพร้อมกัน (anti-reference ตรงจาก PRODUCT.md)
- **Don't** "เหมือนการ์ตูนสำเร็จรูป" — ห้ามใช้ clipart/อิโมจิแทนตัวละคร ห้ามหลุดจากสไตล์ D ของสัตว์ (ยกเว้นอิโมจิเชิงไอคอน UI เช่น 🔊 ที่ใช้อยู่)
- **Don't** "ดูเป็นแอปเรียนจริงจัง" — ห้ามมีคะแนน ดาว เหรียญ แถบความคืบหน้า จับเวลา หรือหน้าตาแบบฝึกหัด/ข้อสอบ
- **Don't** มีสถานะ "ผิด" เชิงลบ — ห้ามใช้สีแดง เสียงหึ่งดุ หรือ X ใหญ่ คำตอบที่ยังไม่ถูกได้แค่ส่ายหัวเบา ๆ แล้วชวนลองใหม่
- **Don't** สร้างเป้าหมายแตะเล็กกว่า 80px สำหรับเด็ก (56px สงวนไว้สำหรับปุ่มผู้ปกครองเท่านั้น)
- **Don't** ผูกเหตุการณ์ mouse/touch แยก — Pointer Events เท่านั้น และห้ามทำให้การเล่นต้องอ่านหนังสือ
