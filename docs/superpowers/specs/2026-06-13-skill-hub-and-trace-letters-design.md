# ชั้นทักษะ + เกมวาดตามรอยตัวอักษร — Design Spec

วันที่: 2026-06-13
สถานะ: อนุมัติแนวคิดแล้ว รอเขียนแผนพัฒนา
ต่อยอดจาก: `2026-06-12-ocean-alaska-game-design.md` (v1 ที่ ship แล้ว)

## เป้าหมาย

ขยาย "ทะเลอลาสก้าของหนู" เป็น**แหล่งรวมเกมพัฒนาการเด็ก 1-15 ปี**:
- **หมวด = ทักษะ** (ผู้ปกครองหาเกมเจอด้วยเป้าหมายพัฒนาการ)
- **ธีมโลก = เครื่องปรุงของแต่ละเกม** สลับหลากหลายเพื่อความน่าสนใจระยะยาว (ทะเลคือโลกแรก ไม่ใช่ข้อจำกัด)
- เพิ่มเกมแรกของโครงใหม่: **วาดตามรอยตัวอักษร** (A-Z ก่อน ระบบรองรับ ก-ฮ ตั้งแต่ต้น)

## ส่วนที่ 1: Hub แบบ "ชั้นทักษะ" (Skill Shelves)

### โครงหน้า

เปลี่ยน hub จาก grid 2×2 ตายตัว เป็น**แถวชั้นวางตามทักษะ เลื่อนแนวตั้งได้** แต่ละชั้นเลื่อนแนวนอนได้เมื่อเกมเกินจอ:

```
[topbar: ชื่อแอป | ปุ่มภาษา(กดค้าง) | ปุ่มเสียง]
🖐️ มือกับตา            ← ป้ายชั้น (แตะแล้วพูดชื่อชั้น)
  [แตะทะเล] [จับคู่เงา] [วาดตามรอย]   ← เลื่อนซ้ายขวาได้
👂 ฟังกับภาษา
  [เสียงเรียกใคร]
🔢 นับกับคิด
  [นับสัตว์ทะเล]
```

### หมวดทักษะตั้งต้น (เพิ่มได้ภายหลังโดยไม่แตะของเดิม)

| id | ป้าย (TH/EN) | ไอคอน | เกมปัจจุบัน | ตัวอย่างอนาคต |
|---|---|---|---|---|
| `hand-eye` | มือกับตา / Hands & Eyes | 🖐️ | แตะทะเล, จับคู่เงา, **วาดตามรอย (ใหม่)** | ระบายสี, ลากเขาวงกต |
| `listen-lang` | ฟังกับภาษา / Listening & Language | 👂 | เสียงเรียกใคร | สะกดคำ, จับคู่คำ-ภาพ |
| `count-think` | นับกับคิด / Counting & Thinking | 🔢 | นับสัตว์ทะเล | บวกลบเลข, puzzle, เกมความจำ |

### ข้อมูลและโค้ด

- ทะเบียนเกมใน `js/hub.js` เพิ่มฟิลด์ `shelf: 'hand-eye'` ต่อเกม + ตาราง `SHELVES` (id, ไอคอน, i18n key) — **เพิ่มเกม/ชั้นใหม่ = แก้ข้อมูล ไม่แก้ logic**
- ป้ายชั้น: ไอคอน + ข้อความสองภาษา (i18n) + แตะแล้วพูดชื่อชั้น (No-Reading Rule)
- การ์ดเกมใช้ `.game-tile` เดิมทุกอย่าง (สามชั้น: สัตว์/ชื่อ/ช่วงวัย + ขอบสี accent) — ความสูงการ์ดคงที่ต่อชั้น (~240px บน iPad, หดตาม media query จอเตี้ยเดิม)
- ธีมโลกสื่อผ่านภาพบนการ์ด ไม่มีโครงสร้าง "โลก" แยกใน v นี้

### การเลื่อน (ข้อควรระวังทางเทคนิค)

- ปัจจุบัน `body { touch-action: none }` ทั้งแอป — **หน้า hub ต้องเปิดการเลื่อน**: container ชั้นวางใช้ `touch-action: pan-y` และแถวเกมใช้ `touch-action: pan-x pan-y`, overflow-y auto / overflow-x auto + `-webkit-overflow-scrolling: touch`
- ฉากเกมยังคง `touch-action: none` เหมือนเดิม (กันเลื่อน/ซูมระหว่างเล่น)
- การแตะการ์ดต้องแยกจากการลากเลื่อน: เก็บพิกัด pointerdown แล้ว launch เฉพาะเมื่อ pointerup ห่างจากจุดกด < 10px (การลากเลื่อนชั้นจะไม่ launch; เบราว์เซอร์ส่วนใหญ่ยิง pointercancel เมื่อ scroll ครอบครอง pointer อยู่แล้ว — เกณฑ์ระยะนี้คือชั้นกันเหนียวข้าม browser)
- คีย์บอร์ด: ป้ายชั้น+การ์ดยังเป็นปุ่ม ผ่าน `onActivate` เดิม; focus เลื่อนตามชั้น

### สิ่งที่ไม่เปลี่ยน

ปุ่มภาษา (กดค้าง), ปุ่มเสียง, ป้ายเตือนเสียงไทย, `?preview=animals`, สถาปัตยกรรม scene `{init, destroy}`

## ส่วนที่ 2: เกมวาดตามรอยตัวอักษร (Trace Letters)

**ชั้น:** มือกับตา · **วัยแนะนำ:** 3-6 ปี · **scene id:** `trace-letters`

### โครงเกม: 2 หน้าจอในฉากเดียว

**จอ 1 — เลือกตัวอักษร (Letter Picker):**
- Grid ตัวอักษร A-Z (26 ปุ่ม `.btn` ≥80px) เลื่อนแนวตั้งได้บนจอเตี้ย (`touch-action: pan-y` เฉพาะ grid)
- เลือกอิสระ ไม่ล็อกลำดับ ไม่มีด่าน (กฎไม่มีแพ้)
- ตัวที่เคยวาดสำเร็จแสดงเป็น**สีเต็ม** (ตัวยังไม่วาด = เส้นขอบ/สีจาง) — ความคืบหน้าแบบสะสมภาพ ไม่มีตัวเลขคะแนน
- ปุ่มสลับชุดอักษร EN/TH โผล่เมื่อมีข้อมูลไทยแล้ว (v1 ซ่อน)
- เปิดจอนี้ครั้งแรก: เสียงพูด "เลือกตัวอักษรที่อยากวาดเลย!"

**จอ 2 — วาดตามรอย (Tracing):**
- ตัวอักษรใหญ่กลางจอเป็น**เส้นประสีเทา** บนการ์ดขาว (`--foam-white`→`--ice-mist`) — ขนาด ~70% ของความสูงจอ
- **นำทีละเส้นตามลำดับขีดจริง:**
  1. เส้นปัจจุบัน: สาธิตอัตโนมัติ 1 รอบ (ลูกศร/ประกายวิ่งตามเส้น ~1 วินาที) + จุดเริ่มเป็นวงกะพริบใหญ่ (≥60px)
  2. เด็กแตะใกล้จุดเริ่ม (รัศมีใจกว้าง ~50px) แล้วลากตามเส้น
  3. ส่วนที่ลากผ่านแล้วเปลี่ยนเป็นเส้นทึบสีสด (สีประจำเกม) + เสียง bubble เบา ๆ เป็นช่วง
  4. นิ้วออกนอกระยะเบี่ยง (~45px จากเส้น): ความคืบหน้าแค่หยุด ไม่มีเสียง/ภาพเชิงลบ กลับเข้าเส้นแล้ววาดต่อได้
  5. ยกนิ้วกลางเส้น: ความคืบหน้าคงอยู่ วางนิ้วต่อจากจุดเดิมได้ (จุด resume กะพริบบอก)
  6. จบเส้น → ding + เส้นถัดไปเริ่มสาธิต; ครบทุกเส้น → ฉลอง
- **ปุ่มช่วย:** กลับ (🏠 บนซ้าย), ดูสาธิตซ้ำ (🔄 บนขวา), ทั้งคู่ผ่าน `onActivate` + aria-label
- **ฉลองเมื่อครบตัว:** ตัวอักษรเติมสีเต็ม + boing + cheer + คอนเฟตตี + เสียงอ่าน "เอ! A — Anchor, สมอเรือ" (ภาษาปัจจุบันนำ อีกภาษาตาม ตามแบบ `speakName`) + รูปคำศัพท์ pop-in (ถ้ามี art) → ค้าง ~2.5 วิ แล้วกลับจอเลือกอัตโนมัติ พร้อมตัวที่เพิ่งวาดเด้งเป็นสีเต็ม

### โมเดลข้อมูล (รองรับ ก-ฮ ตั้งแต่ต้น)

```
js/games/data/letters-en.js   — ชุด A-Z (v1)
js/games/data/letters-th.js   — ชุด ก-ฮ (อนาคต ใช้ engine เดิม)

ต่อหนึ่งตัวอักษร:
{
  char: 'A',
  name: { th: 'เอ', en: 'A' },              // เสียงเรียกชื่อตัวอักษร
  word: { th: 'สมอเรือ', en: 'Anchor' },     // คำศัพท์ประกอบ
  art: (w) => SVGElement | null,             // รูปคำศัพท์ (optional — ดูนโยบาย art)
  viewBox: '0 0 100 120',
  strokes: [ 'M 10 110 L 50 10', 'M 50 10 L 90 110', 'M 28 70 L 72 70' ],
  // ลำดับใน array = ลำดับขีด; ทิศของ path = ทิศการลาก
}
```

- **นโยบาย art:** ทุกตัวมี `word` + เสียงพูดเสมอ; รูปเป็น optional — เฟสแรกใส่ art เฉพาะตัวที่จับคู่สัตว์/ของที่มีอยู่ได้ (เช่น O=Orca, P=Puffin, S=Salmon, C=Crab) ตัวที่ยังไม่มีรูป ใช้ตัวอักษรเต้น + คอนเฟตตีแทน แล้วทยอยวาดเพิ่ม — **ห้ามใช้ clipart/อิโมจิแทนรูปคำศัพท์** (anti-reference เดิม)
- คำศัพท์ชุดแรกเลือกโทนทะเล/ธรรมชาติให้เข้าจักรวาลเดิมก่อน

### Engine การตรวจการลาก (pure logic แยกทดสอบได้)

`js/games/logic/trace-logic.js` — ฟังก์ชัน pure ทั้งหมด ทดสอบด้วย `node --test`:

- ฉาก sample จุดบนเส้น (`getPointAtLength` ทุก ~6px — DOM-dependent ทำในฉาก) แล้วส่ง `points[]` เข้า logic
- `advance(points, currentIndex, finger, tolerance)` → index ใหม่: เดินหน้าได้เมื่อนิ้วอยู่ในรัศมี tolerance ของจุดถัดไป (กิน sample ได้หลายจุดต่อ move เพื่อรองรับลากเร็ว) — **บังคับทิศโดยธรรมชาติ** เพราะต้องกินจุดตามลำดับ ลากย้อนทิศ = ไม่คืบหน้า เฉย ๆ
- `isNearStart(points, finger, startRadius)` → เริ่มเส้นได้ไหม
- `isComplete(points, index, endSlack)` → จบเส้น (ยอมขาดปลาย ~2 จุดท้าย กันเด็กยกนิ้วก่อนสุดเส้นนิดเดียว)
- ค่าตั้งต้น: `tolerance = 45px`, `startRadius = 50px` (สเกลตามขนาด render จริง)

### ความคงทน (ตามแบบแผนโปรเจกต์)

- Pointer Events เท่านั้น + `setPointerCapture` + `pointercancel` = ยกนิ้ว (เก็บ progress)
- นิ้วที่สองระหว่างวาด: เพิกเฉย (guard แบบ shadow-match)
- timers ทุกตัวเข้า `_timers[]` ล้างใน `destroy()` + `speechSynthesis.cancel()` ใน destroy
- หมุนจอ/resize ระหว่างวาด: re-render ตัวอักษรปัจจุบัน รีเซ็ตเฉพาะเส้นที่กำลังวาด (เส้นที่จบแล้วคงอยู่)
- บันทึกความคืบหน้า: localStorage `oceanAlaska.traceDone.en = ['A','B',...]` ผ่าน settings.js (มี in-memory fallback เดิม)
- `prefers-reduced-motion`: สาธิตเส้นเปลี่ยนจากอนิเมชันวิ่งเป็นไฮไลต์เส้นปัจจุบันนิ่ง ๆ + จุดเริ่มเด่น

### การทดสอบ

- `tests/trace-logic.test.mjs`: advance ปกติ/ลากเร็ว/ออกนอกเส้น/ย้อนทิศ/resume, isNearStart, isComplete พร้อม endSlack
- Playwright: เปิดเกมจริง วาด A ด้วย pointer events จำลองครบทุกเส้น → ตรวจฉลอง + สีเต็มใน picker + บันทึก localStorage; ตรวจจอ 1180×820 และ 844×390
- เทสต์เดิม 16 ตัวต้องผ่านครบ

## นอกขอบเขตรอบนี้ (บันทึกไว้)

- ข้อมูล ก-ฮ + ลำดับขีดไทย (engine รองรับแล้ว เหลือทำ data)
- ตัวพิมพ์เล็ก a-z, ตัวเลข 0-9 (ใช้ engine เดิมได้ทันที)
- art คำศัพท์ครบ 26 ตัว (ทยอยเพิ่ม)
- โหมดโลกธีม/เปลี่ยนธีมต่อเกม (วิสัยทัศน์ระยะยาว)
- เกมวัย 6-15 อื่น ๆ (สะกดคำ บวกลบ puzzle — เพิ่มเป็นเกมที่ 6,7,... บนชั้นเดิม)
